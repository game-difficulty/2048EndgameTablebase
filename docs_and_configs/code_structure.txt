2048 Endgame Tablebase - current code structure

1. Runtime entrypoints
- backend_server.py
  Thin FastAPI/WebSocket entrypoint for the desktop app.
- backend/
  Main backend package used by the Vue frontend.
- frontend/
  Vue application source and build output.

2. Backend layout
- backend/handlers/
  WebSocket action handlers grouped by feature.
- backend/minigames/
  Minigame registry, engine, services, sessions, and migrated game variants.
- backend/analysis_core.py
  Qt-free analysis logic extracted from the old desktop implementation.
- backend/session.py, backend/state.py, backend/serialization.py
  Shared backend session/state/serialization utilities.

3. Frontend layout
- frontend/src/app/
  App shell, top-level tabs, shared app settings.
- frontend/src/features/
  Feature-based Vue pages and components.
- frontend/src/components/
  Shared UI components reused across multiple features.
- frontend/src/utils/ and frontend/src/services/
  Shared frontend utilities and transport helpers.

4. Core algorithm packages
- egtb_core/
  Python-side tablebase, replay, compression, and solver logic.
- ai_and_sort/
  Merged native workspace for compiled helpers.
  - src/
    C++ source for ai_core and mover_core.
  - include/
    Shared C++ headers.
  - x86simdsort/x86-simd-sort/
    Modified SIMD sorting source kept in-repo for self-builds.
  - CMakeLists.txt, make.sh, README.md
    Native build entrypoints and build guidance.

5. Root-level application files
- Config.py
  Global config loader/saver and application-level settings integration.
- SignalHub.py
  Shared signal/event helper still referenced by runtime code.
- error_bridge.py
  Frontend/global error reporting bridge.
- build_linux.sh
  Linux helper script that builds the frontend and native workspace, then optionally launches the app.
- requirements.txt
  Main Python dependencies list.

6. Documentation and configuration
- docs_and_configs/
  Centralized directory for docs and runtime configuration.
  - config
    Local runtime config/state file.
  - mistakes_book.pkl
    Local notebook persistence.
  - patterns_config.json, default_patterns.json, themes.json
    Pattern and theme definitions.
  - help/
    User-facing help files.
  - code_structure.txt, github_upload_guide.md, packaging_guide.md
    Repository and distribution documentation.

7. Assets and static resources
- pic/
  Shared static assets.
- minigames/pic/
  Minigame assets and covers.
- font/
  Fonts.
- mathjax/
  Help/document rendering dependency.
- favicon.ico
  App/browser icon.

8. Minigame compatibility layer
- minigames/MinigameMover.py
  Legacy mover module still reused by backend/minigames/engine/mover.py.

9. Tools
- tools/AItest.py
  Standalone native-module smoke test.
- tools/Analyzer2.py
  Auxiliary analysis script, not part of runtime startup.

10. Local-only / generated content
- frontend/node_modules/, frontend/dist/
  Frontend dependency/build output.
- ai_and_sort/build/
  Native extension build output.
- ai_and_sort/x86simdsort/x86-simd-sort/builddir/
  SIMD sort build output.
- __pycache__/ and temporary compile artifacts
  Python-generated cache files.
